## 2. 这场比赛在做什么

这是一个时间序列预测任务。

你需要根据历史数据，预测厄瓜多尔连锁超市 `Corporacion Favorita` 在未来 16 天内的商品销量。

更具体地说：

- 每一行数据对应某一天、某个门店、某个商品类别
- 目标列是 `sales`
- 你需要对 `test.csv` 中每一行给出预测值

一句话理解：

这道题本质上是在做“根据过去，预测未来每天卖多少”。

## 3. 数据长什么样

### `train.csv`

主要字段：

- `date`：日期
- `store_nbr`：门店编号
- `family`：商品类别
- `sales`：当天销量，这是预测目标
- `onpromotion`：当天促销商品数

### `test.csv`

和训练集很像，但没有 `sales`，因为这正是你要预测的内容。

测试集日期范围是：

- `2017-08-16` 到 `2017-08-31`

也就是需要预测未来 16 天。

### `stores.csv`

门店基础信息：

- `city`：城市
- `state`：州
- `type`：门店类型
- `cluster`：门店分群

### `transactions.csv`

记录每天每个门店的交易次数。

它可以帮助你理解“客流”和“销量”之间的关系。

### `oil.csv`

记录石油价格。

因为这家超市所在国家的经济环境可能受到油价影响，所以油价可能和销量有关。

### `holidays_events.csv`

记录节假日、地区庆典和特殊事件。

节假日通常会影响销量，比如：

- 节前备货增加
- 节日期间某些品类销量暴涨
- 不同城市和地区影响不一样

## 5. 这题真正考你什么

这题不是只考“套一个模型”。

它真正考的是：

- 你能不能把时间序列问题整理成可建模的数据
- 你能不能设计合适的时间特征
- 你能不能处理节假日、促销、趋势、季节性
- 你能不能用合理的方式做验证，而不是把未来信息泄露给模型

## 6. 如果你想把它当作“神经网络练习作业”

如果你现在主要会的是基础神经网络，而不是传统机器学习模型，那么这道题完全可以改造成一个很好的神经网络练习项目。

你可以把它理解成：

- 输入：某一天、某个门店、某个商品类别、促销情况等信息
- 输出：当天销量 `sales`

从神经网络角度看，这其实是一个**回归问题**，不是分类问题。

也就是说：

- 不是预测“属于哪一类”
- 而是预测“一个连续数值”，也就是销量

### 为什么这题适合练神经网络

因为它可以让你顺手练到这些基础概念：

- 如何把表格数据变成神经网络输入
- 如何处理类别特征
- 如何做回归任务
- 如何设计输入层、隐藏层、输出层
- 如何选损失函数
- 如何做训练集 / 验证集划分
- 如何避免时间序列里的“未来信息泄露”

### 这题用神经网络时，你可以怎么理解模型

最基础的神经网络版本可以这样想：

- 输入层：放入你整理好的特征
- 隐藏层：学习“日期、门店、品类、促销”和销量之间的非线性关系
- 输出层：输出一个实数，表示预测销量

这里最关键的一点是：

**输出层只有 1 个神经元。**

因为你不是在做多分类，而是在预测一个具体的销量数值。

### 最适合你的模型选择

如果你是拿这题练基础神经网络，最推荐的是从下面这个顺序开始：

1. 最基础的 `MLP` / 全连接神经网络
2. 再考虑给类别特征加 `Embedding`
3. 最后再考虑更复杂的时间序列特征

也就是说：

不要一上来就做 LSTM、Transformer。

因为这题首先难的不是网络结构，而是：

- 数据整理
- 特征构造
- 时间切分
- 验证方式

## 7. 推荐你怎么完成作业：神经网络版 1.0 -> 2.0

下面我给你一个非常适合作业提交的渐进路线。

### 1.0 版本：先做一个最基础的神经网络 baseline

目标：

- 先完整做出一个“能训练、能验证、能提交”的神经网络版本
- 不追求复杂结构，重点是把神经网络流程真正走通
- 让你能用自己会的内容完成这份作业

建议步骤：

1. 读入 `train.csv` 和 `test.csv`
2. 把 `date` 转成日期格式
3. 从 `date` 提取基础时间特征：
   - 年
   - 月
   - 日
   - 星期几
   - 是否周末
4. 只用这些最基础特征，再加上：
   - `store_nbr`
   - `family`
   - `onpromotion`
5. 把这些特征整理成神经网络输入：
   - 数值特征可以直接输入
   - 类别特征可以先做整数编码，或者 one-hot
6. 建一个最基础的 `MLP`
   - 输入层
   - 1 到 2 层隐藏层
   - 输出层 1 个神经元
7. 做回归训练
   - 目标是预测 `sales`
   - 可以考虑先对 `sales` 做 `log(1+x)` 变换，让训练更稳定
8. 划分验证集：
   - 不要随机切分
   - 用时间切分
   - 例如训练到 `2017-07-31`，验证 `2017-08-01` 到 `2017-08-15`
9. 在验证集上评估效果
10. 对 `test.csv` 预测，并生成提交文件

这个版本最重要的不是分数，而是你要真的明白：

- 输入是什么
- 标签是什么
- 神经网络为什么能做这个回归任务
- 为什么时间序列不能随机打乱切分

这个版本的关键词是：

- 神经网络入门版
- 先跑通
- 先理解结构
- 先完成作业

### 1.0 版本里，建议你用什么网络结构

你完全可以用最朴素的结构：

- 输入层
- 全连接层 `Linear`
- 激活函数 `ReLU`
- 全连接层 `Linear`
- 输出层 `Linear(?, 1)`

比如这种思路：

- 第一层：学基础组合关系
- 第二层：学更复杂的非线性关系
- 最后一层：输出一个销量值

### 1.0 版本里，建议你怎么解释给老师听

你可以这样写：

“本实验将零售销量预测问题建模为监督学习中的回归任务。输入特征包括日期信息、门店编号、商品类别和促销数量，输出为当天销量。模型采用多层感知机（MLP）进行拟合，通过隐藏层学习输入特征与销量之间的非线性映射关系，并采用时间顺序划分训练集与验证集，以避免未来信息泄露。”

### 1.0 版本里，损失函数怎么理解

如果你用神经网络做回归，最常见的是：

- `MSELoss`
- `L1Loss`
- `SmoothL1Loss`

对你来说，最简单的是先用 `MSELoss`。

你可以这样理解：

- 预测值和真实值差得越远，损失越大
- 训练的目标就是让这个损失越来越小

### 1.0 版本里，为什么可以先不做复杂时间模型

因为这道题虽然是时间序列任务，但你当前是拿它练基础神经网络。

所以第一版只要做到：

- 从日期中提取时间特征
- 把问题转成表格回归
- 用 MLP 学习映射关系

这已经是一个合理的“神经网络作业”了。

### 1.5 版本：让它更像一个完整的神经网络作业

目标：

- 在基础 MLP 之上，体现你对时间序列和特征工程有进一步理解
- 让作业内容更扎实，而不是只停留在“普通表格回归”

可以增加这些内容：

1. 加入更多特征
   - `stores.csv`
   - `transactions.csv`
   - `oil.csv`
   - `holidays_events.csv`

2. 增加滞后特征 `lag`
   - 前 1 天销量
   - 前 7 天销量
   - 前 14 天销量

3. 增加滚动统计特征
   - 过去 7 天平均销量
   - 过去 14 天平均销量
   - 过去 28 天平均销量

4. 更合理地处理类别特征
   - 先用 one-hot
   - 或升级为 embedding

这个阶段你会开始真正接触到：

- 为什么特征工程对神经网络也很重要
- 为什么时间序列不是只靠“深模型”就能解决
- 为什么过去销量本身往往比静态特征更重要

### 2.0 版本：做出一个更像深度学习项目的方案

目标：

- 让这份作业不只是“我会搭 MLP”
- 而是体现出你已经开始理解“如何把神经网络用于真实预测任务”

你可以考虑往这些方向升级：

1. 把 `store_nbr` 和 `family` 做成 `Embedding`
   - 这会比简单整数编码更合理
   - 也更符合神经网络处理类别特征的常见方式

2. 使用更好的输入特征
   - 周期特征
   - 节假日特征
   - 油价特征
   - lag 和 rolling 特征

3. 做更严格的验证
   - 用 rolling validation
   - 或 walk-forward validation

4. 做模型分析
   - 不同门店预测误差对比
   - 不同品类预测误差对比
   - 节假日前后销量变化分析

5. 讨论神经网络的优缺点
   - 优点：可以学习复杂非线性关系
   - 缺点：对特征工程和训练细节比较敏感

### 如果老师问你：为什么不用 CNN、RNN、LSTM

你可以这样回答：

可以回答：

“是的。虽然模型结构比较基础，但只要使用了多层神经网络、前向传播、损失函数和反向传播训练参数，本质上就属于神经网络 / 深度学习方法。只是这里采用的是适合初学者实现和分析的基础版本，而不是更复杂的序列网络结构。”
